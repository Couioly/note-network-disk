
>[!faq]- JVM、JRE、JDK有什么联系和区别？
>- JVM：指可以运行Java字节码的虚拟计算机，是JRE的一部分。
>- JRE：面向Java程序的使用者，提供了Java应用程序的运行环境。JRE包括JVM和Java核心类库和支持文件。
>- JDK：面向Java程序的开发者，是整个Java的核心，包括了JRE、Java工具和Java基础的类库。

>[!faq]- 软件版本是不是越新越好？
新的软件版本带来了新的更加丰富的功能，但是当业界还没有在足够多的项目中应用、测试该软件之前，新版本也可能带来潜在的风险。对于Java初学者来说，追求新的版本更没有必要，除非要学习Java的一些新的语言特性。当然，软件版本也不能太旧。

>[!faq]- 操作系统、JDK与IDEA的版本匹配问题？
软件之间要彼此搭配。在此，操作系统、JDK和IDEA IDE肯定要搭配一致的，否则就会出问题。因此在下载安装文件之前要确定操作系统。

 >[!faq]- Path环境变量的作用？
 Path环境变量是用于保存一系列可执行文件路径的变量，这些路径指定了系统可以在其中查找可执行文件的位置。当在Windows系统的命令行中运行一条可执行命令时，系统首先会在当前目录中寻找该命令对应的可执行文件。如果在当前目录中未找到，系统会继续在Path环境变量中定义的路径中搜索对应的可执行文件。

>[!faq]- 如何从键盘上输入一个char类型的数据？
经过查询JDK文档，Java没有提供从键盘输入char类型数据的方法。但可以用char ch=sc.next().charAt(0);间接实现。

>[!faq]- 什么是Unicode编码？
Unicode是国际组织制定的可以容纳世界上所有文字和符号的字符编码方案。编码长度为16Bits，可以容纳65536个字符。前256个：128个标准ASCII码，128个扩展ASCII码。然后是希腊字母、西里尔文、亚美尼亚文、希伯来文等拼音文字。从0X3000到0X9FFF是汉语、韩语、日语等象形文字。
