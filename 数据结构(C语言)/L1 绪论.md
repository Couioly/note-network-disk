## 1 数据结构的基本概念
---

![[Pasted image 20250917204959.png]]

观察上述表格的特征，表格每一行是一个由序号、用户名、电话号码和用户地址等项组成用户的有关信息。

- **基本项**：有独立意义的最小标识单位，如表中的序号、用户名和电话号码等项；
- **组合项**：由一个或多个基本项或组合项组成且具有独立意义的标识单位，如表中的用户地址；
- **字段**：指每一个基本项或组合项；
- **节点**：由若干个字段构成，如表中每一行称为一个节点；
- **关键码**：能唯一标识一个节点的字段或几个字段的组合，如表中的序号字段。

### 1.1 数据的逻辑结构

**表首节点**：前面没有其他节点且后面有一个和它相邻的节点
**表尾节点**：后面没有其他节点且前面有一个和它相邻的节点

除了上述两个节点外，表中的其他节点都有且仅有一个和它相邻的位于它之前的节点，也有且仅有一个和它相邻的位于它之后的节点。这就是用户信息表的逻辑结构。

>[!info] 逻辑结构
>数据元素间抽象化的相互关系，与数据的存储无关，独立于计算机，它是从具体问题抽象出的数学模型。

**数据的逻辑结构**：数据和数据之间所存在的逻辑关系，可以使用一个二元组表示：

```
B = (K,R)
```

- `K`：指数据，即节点的有限集合
- `R`：指集合K上关系的有限集合（关系是指从集合K到集合K的关系）

**示例**：

有5人分别记作a、b、c、d、e，其中a是b的父亲，b是c的父亲，c是d的父亲，d是e的父亲，如果只讨论它们之间存在的父子关系，则可以用下面的二元组形式化的表示：

```
B = (K,R)
```

其中：

```
K = {a,b,c,d,e}
R = {r}
r = {<a,b>,<b,c>,<c,d>,<d,e>}
```

若ki∈K，kj∈K，<ki,kj>∈r：
- ki是kj的相对于关系r的前驱节点
- kj是ki的相对于关系r的后驱节点

**开始节点**：节点没有前驱节点；
**终端节点**：节点没有后驱节点；
**内部节点**：既不是开始节点也不是终端节点。

#### 划分方法一

**线性结构**：有且仅有一个开始节点和一个终端结点，而其他的每一个节点有且仅有一个前驱和一个后继。（例如：线性表、栈、队列、串）

**非线性结构**：一个结点可能有多个直接前趋和直接后继。（如集合结构、树状结构和图形结构）

#### 划分方法二

- **集合结构**：数据之间不存在任何关系
- **线性结构**：数据之间存在一对一的关系
- **树状结构**：数据之间存在一对多的关系（即它存在一个开始节点，有多个终端节点，除开始节点外每一个节点有且仅有一个前驱）
- **图形结构**：数据之间存在多对多的关系（即每个节点都可以有多个前驱和后继）

### 1.2 数据的存储结构

一个数据结构 `B=(K,R)`，必须建立从节点集合到计算机某个存储区域M的一个映像，这个映像通常要直接或间接的表达节点之间的关系R。

>[!info] 存储结构（物理结构）
>数据元素及其关系在计算机存储器中的存储方式。

#### 顺序存储

将逻辑相邻的节点存储在连续存储区域M的相邻存储单元中，使逻辑相邻的节点一定是物理相邻的。顺序存储常用于存储**具有线性结构**的数据。

>[!info]
>该映像是通过物理上存储单元的相邻关系来体现节点间相邻的逻辑关系，通过相对位置描述数据关系

**示例**：

对于一个数据结构 `B=(K,R)`，其中：

```
K = {k₁,k₂,k₃,k₄,k₅}
R = {r}
r = {<k₁,k₂>,<k₂,k₃>,<k₃,k₄>,<k₄,k₅>}
```

顺序存储示意图：

![[Pasted image 20250918184832.png]]

#### 链式存储

**链式存储方式**：给每个节点附加一个指针段，一个节点的指针所指的是该节点的后继存储地址（若节点存在多个后继，则可以使用多个指针）。

>[!info]
>该映像是通过**指针**来实现数据的逻辑结构的

**示例**：

数据的逻辑结构 `B=(K,R)`，其中：

```
K = {k₁,k₂,k₃,k₄,k₅}
R = {r}
r = {<k₁,k₂>,<k₂,k₃>,<k₃,k₄>,<k₄,k₅>}
```

数据的逻辑结构 `B=(K,R)`，其中：

```

R = {r}
r = {<k₁,k₂>,<k₁,k₃>,<k₂,k₄>,<k₄,k₅>}
```

#### 索引存储

存在一个线性结构，若开始节点的索引号为1，其他节点的索引号等于其前驱节点的索引号`+1`，则每个节点都有唯一的索引号。

>[!info] 索引存储
>指根据节点的索引号确定该节点的存储地址

#### 散列存储

**散列存储的思想**：构造一个从集合K到存储区域M的函数h，该函数的定义域为K，值域为M，K中的每个节点kᵢ在计算机中的存储地址由h(kᵢ)确定。

>[!info] 数据结构的存储密度
>把数据本身所占存储空间的大小与整个数据结构所占存储空间的大小之比称为数据结构的存储密度。
>- 数据结构的存储密度不大于1；
>- 顺序存储的存储密度为1；
>- 链式存储的存储密度小于1。

### 1.3 数据的运算集合

**为一批数据定义的所有运算(或操作)** 构成一个运算（或操作）集合。

>[!info]
>对于一批数据，数据的运算是定义在数据的逻辑结构之上的，而运算的具体实现依赖于数据的存储结构。

数据的运算结合要视情况而定，一般而言，数据的运算包括插入、删除、检索、输出和排序等。

- **插入**：指在一个结构中增加一个新的节点
- **删除**：指在一个结构中删除一个节点
- **检索**：指在一个结构中查找满足条件的节点
- **输出**：指将一个结构中所有节点的值输出
- **排序**：指将一个结构中所有节点按某种顺序重新排序

**栈和队列**

![[Pasted image 20250918213740.png]]

**数据结构**：指按一定的逻辑结构组成一批数据，使用某种存储结构将这批数据存储于计算机中，并在这批数据上定义一个运算集合。

>[!info] 同理
>也就是说，只有给定一批数据的逻辑结构和它们在计算机中的存储结构，并且定义数据的运算集合，才能确定一个数据结构。

## 2 数据类型和抽象数据类型
---
**数据的抽象经历了3个发展阶段**：

- 第一阶段：从无类型的二进制数到基本数据类型的产生；
- 第二阶段：从基本数据类型到用户自定义类型的产生；
- 第三阶段：从用户自定义类型到抽象数据类型的产生。

### 2.1 数据类型

**数据类型**：反映数据的取值范围以及对这类数据可以施加的运算。（某个变量的数据类型是指该变量所有可能的取值集合）

**数据类型包括的两个方面**：
- 数据属性
- 在这些数据上可以施加的运算集合

### 2.2 抽象数据类型

**抽象数据类型**是基本数据类型的延伸和发展，是与**表示**无关的数据类型，是一个数据模型及定义在该模型上的一组运算。

>[!info]
>定义一个抽象数据类型时，必须给出它的名字及各运算的运算符名，即**函数名**，并且规定这些函数的参数性质。

### 2.3 抽象数据类型的描述和实现

**抽象数据类型的描述**：
- 抽象数据类型的名称
- 数据的集合
- 数据之间的关系和操作的集合

>[!tips] 描述抽象数据类型
>- **抽象数据类型的名称**：任意给定，但最好意思贴近；
>- **数据的描述**：给出数据的集合以及这些数据之间关系的描述；
>- **操作的描述**：给出每个操作的名称、操作的前置和后置条件以及操作的功能。

**抽象数据类型的操作分为**：

- **构造操作集**：是一个抽象数据类型中必须定义的操作集合，抽象数据类型中的数据元素均可以通过它们组合操作获得；
- **非构造操作集**：为了方便使用抽象数据类型而定义的一些操作的集合。

**抽象数据类型描述的一般形式如下**：

```cpp
ADT 抽象数据类型名称 {
	数据对象:
			...
	数据关系:
			...
	操作集合:
		操作名1:
			...
		操作名n:
}ADT 抽象数据类型名称
```

抽象数据类型的具体实现依赖程序设计语言。面向对象程序设计语言中的类支持抽象数据类型，也支持信息隐藏。

## 3 算法和算法分析
---
### 3.1 算法的基本概念及特征

**算法**：求解问题的一种方法或一个过程。严格地讲，它是由若干条指令组成地有穷序列。它既可以用自然语言描述，也可以用高级程序设计语言或伪代码描述。

简单理解：为了求解某个问题，必须给出一系列的运算规则，这一系列的运算规则是有限的，表达了求解问题的方法和步骤，这就是一个**算法**。

>[!tips] 算法的5大特征
>1. **有穷性**：算法的执行必须在有限步内结束；
>2. **确定性**：算法的每一步骤必须是确定的、无二义性的；
>3. **输入**：算法可以有0个或多个输入；
>4. **输出**：算法一定有输出结果；
>5. **可行性**：算法中的运算都必须是可以实现的。

**程序和算法的主要区别**：算法具备**有穷性**，而程序不需要具备该特征。

### 3.2 时间复杂度和空间复杂度

**一个算法的优劣主要从两个方面来衡量**：
- 算法执行时间
- 所需要占用的存储空间

其中，**算法执行时间并非算法执行的绝对时间**，而是采用算法**执行过程中其基本操作的执行次数**，即**计算量**来度量。

算法基本操作的执行次数一般是与问题规模有关的，对于节点个数为n的数据处理问题，用T(n)表示算法基本操作的执行次数。

**以`O`来表示算法的时间复杂度与空间复杂度**：设f(n)与g(n)是定义在正整数集合上的两个函数，如果存在两个正常数c和n₀，对于所有的`n>=n₀`，有`|f(n)|<=c*g(n)`则记作`f(n)=O(g(n))`。也就是说对几乎所有的n值，函数f(n)以函数g(n)为上界。

**示例1**：

```c
{
++x;s=0;
}
```

将x自增看成是基本操作，则语句频度为1，即时间复杂度为`O(1)`，若将`s=0`也看成基本操作，则语句频度为2，其时间复杂度仍为`O(1)`，即**常量阶**。

**示例2**：

```c
for (i=1; i<=n; ++i)
	{++x; s+=x;}
```

语句频度为`2n`， 其时间复杂度为`O(n)`，即时间复杂度为**线性阶**。

**示例3**：

```c
for (i=1; i<=n; ++i)
	for (j=1; j<=n; ++j)
		{++x; s+=x;}
```

语句频度为：`2n²`，其时间复杂度为：`O(n²)`，即时间复杂度为**平方阶**。

按数量级递增排序，常见算法的时间复杂度有常数阶O(1)、对数阶O(log₂n)、线性阶O(n)、线性对数阶O(nlog₂n)、平方阶O(n²)、立方阶O(n³)、...、k次方阶O(nᵏ)、指数阶O(2ⁿ)，即：

```
O(1) < O(log₂n) < O(n) < O(nlog₂n) < O(n²) < O(n³) < ... < O(nᵏ)< O(2ⁿ)
```

>[!warning] 注意
>算法的时间复杂度**不仅与问题的规模有关**，还**与问题的数据的初始状态有关**。对于在一个线性结构中的检索问题，设问题规模为n，算法采用从左向右的方向检索，若检索的x正好就在线性结构的第一个节点，则检索的时间复杂度为O(1)；若x不在线性结构中，或x等于线性结构中的最后一个节点的值，则检索时间复杂度为O(n)。因此便得出以下概念：
>> [!tips] 新的概念
>> **算法在最好情况下的时间复杂度**：指算法计算量的最小值
>> **算法在最坏情况下的时间复杂度**：指算法计算量的最大值
>> **算法在平均情况下的时间复杂度**：指算法在所有可能的情况下的计算量经过加权计算出的平均值

**评价算法要考虑以下几个方面**：

- <font color=red>正确性</font>（有限时间内得到正确结果）
- 高效性（时间复杂度和空间复杂度）
- 可读性（理解、调试、修改）
- 稳健性（处理非法数据）

>*例：学生表 > 个人记录 > 学号、姓名...*

5.数据对象的实例如下：
- 整数数据对象  N = {0,∓1，∓2，...}
- 学生数据对象  学生记录的集合

6.数据结构是带“结构”的数据元素的集合，“结构”就是指数据元素之间存在的关系。

7.时间复杂度的渐进表示法

算法中基本语句重复执行的次数是问题规模n的某个函数f(n)，算法的时间量度记作：`T(n)=O(f(n))`

>表示随着n的增大，算法执行的时间的增长率和f(n)的增长率相同，称**渐进时间复杂度**

8.分析算法时间复杂度的基本方法：

- 找出**语句频度最大**的那条语句作为**基本语句**
- 计算**基本语句**的频度得到问题规模n的某个函数f(n)
- 取其**数量级**用符号`O`表示时间复杂度

**示例**：

求出下述代码的时间复杂度：

```c
x=0;y=0;
for(int k=0;k<n;k++)
	x++;
for(int i=0;i<n;i++)
	for(int j=0;j<n;j++)
		y++;
```

f(n) = n²+n   ==> T(n) = O(n²)

将一维数组a中的n个数逆序存放到原数组中：

```c
"""原地工作 S(n)=O(1)"""
for(i=0;i<n/2;i++){
	t=a[i];
	a[i]=a[n-i-1];
	a[n-i-1]=t;
}
```

```c
"""S(n)=O(n)"""
for(i=0;i<n;i++)
	b[i]=a[n-i-1];
for(i=0;i<n;i++)
	a[i]=b[i];
```
